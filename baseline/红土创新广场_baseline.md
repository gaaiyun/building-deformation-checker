## 八、我的核对方法说明

1. **数据源**：将 22 个 OCR `.txt` 全部读入，按页定位每张数据表。OCR 输出按页分隔，文本质量随页面复杂度变化大（page 12/14/16/17 几乎空白；page 18 出现 200+ 行重复污染）。
2. **结构识别**：先用关键词（"基坑顶位移监测数据成果表"、"测斜观测数据成果图表"、"点号"、"深度"）锚定表头，再按"本次累计 / 上次累计"两列做主轴提取。**优先信任本次/上次累计两列**，因为 OCR 把 "本期变化"列分散到不同行。
3. **公式独立重算**：用 Python 脚本（`_audit_calc.py`）按下列公式独立计算并与 OCR 的"本期变化"列比对：
   - 本期变化 = 本次累计 − 上次累计
   - 速率 = 本期变化 / 14 d（14 d 来自测斜表"第 590 次 2019/9/14 → 第 591 次 2019/9/28"，与封面 2019.9.17 ～ 10.1 一致）
4. **容差**：累计 ±0.15 mm；速率 ±0.05 mm/d；累计 >10 mm 时按 5% 放宽。
5. **方向规则**：深层位移所有有效值均为负（向坑内位移），故 "正向最大" 取最接近 0（杆底 0.00）。"负向最大" 取最负数。
6. **全局极值**：跨 5 个 OCR 抓到的孔位汇总极值；提示 CX10 缺失。
7. **简报核对**：对照 page 6 简表与 page 7 文字结论中的每个具体测点与具体数值，逐项与重算结果对照。
8. **未覆盖项**：OCR 未提供水位数据表、CX10/CX01-08/CX11/CX14/CX15/CX17/CX19/CX20 测斜表、立柱/支撑/桩内力等"本期 0 点"项目均按工作量表确认为不监测。
9. **OCR 偏差归类**：所有 OCR "本期变化"列与重算值偏差均 ≤ 0.16 mm（在 ±0.15 容差或临界），未发现数值实质性错误；偏差源于多列融合或行错位。
10. **环境**：Python 3.13 / Windows 11，UTF-8 输出，所有数值以浮点保留 3 位。
