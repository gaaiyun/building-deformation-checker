# 异构 PDF 表格核对流程设计

## 目标

不同监测报告的表头、单位、跨页方式和统计口径会变化。系统不能把“识别出数字”直接等同于“可以硬计算核对”，必须先判断字段是否齐全、单位是否同口径、统计口径是否明确。

## 推荐流程

1. OCR/文本提取
   - 默认先用文本层；文本质量差或扫描件时启用 PaddleOCR 异步 API。
   - OCR 输出保留 raw/clean/debug 三份，便于追溯误报来源。

2. 表格切分与跨页合并
   - 以监测项、测孔编号、表头字段集合、页序作为合并键。
   - 普通表缺测孔编号时按监测项合并；深层位移按监测项 + 测孔编号合并。
   - 续表只补数据行，不重复统计行。

3. 字段语义标准化
   - 将不同叫法映射到标准字段：初始值、本次值、上次累计、本次累计、本次变化、累计变化、变化速率、安全状态。
   - 单位从表头、单位行和数据特征三处共同判断。
   - LLM 可补充判断，但不能覆盖数据证据明确的单位转换规则。

4. 可核对性判定
   - 硬校验：字段齐全、单位明确、公式口径明确。
   - 弱校验：字段基本齐全但基准不可靠、单位需转换、统计口径存在行业惯例。
   - 跳过并说明：缺核心字段、跨页归属不确定、只有汇总无明细、原文 OCR 明显错列。

5. 专项计算器
   - 普通位移：累计变化 = 本次值 - 初始值；变化速率 = 本次变化 / 监测间隔。
   - 高程/沉降：初始/本次通常为 m，变化量通常为 mm，需乘以 1000；若基准不可靠只做 warning/info。
   - 深层位移：按测孔分组；速率可按绝对值比较，跨孔汇总需谨慎。
   - 锚索/支撑：内力类不要求变化速率；累计变化 = 本次内力 - 初始内力。
   - 水位：初始基准和方向约定常变化，容差和严重性降低。

6. 统计与汇总核对
   - 正/负方向最大值要区分方向，不简单按绝对值。
   - 全正或全负时，报告可能填“最小正值/最接近 0 的负值”，应降为 info。
   - 最大速率既可能按绝对值最大，也可能按数值最大；同号速率时需要接受报告口径并说明。

7. 报告输出
   - 明确列出“已硬校验”“弱校验”“跳过原因”。
   - 每个问题标注疑似来源：报告原文、OCR/列映射、规则口径、LLM 解析。

## 速度策略

- OCR 只在需要时启用，结果写入 debug/cache 目录，重复实验复用。
- LLM 只负责结构化和语义补强，公式计算全部由本地规则完成。
- 表格分析计划先用规则生成，只有发现单位/基准异常时再调用 LLM 增强。
- 自验证只复核 error 级问题；没有 error 时跳过。
- 需要更快响应时使用 MiniMax-M2.7-highspeed；需要更稳结构化时使用 MiniMax-M2.7。
