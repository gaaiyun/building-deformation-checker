## 5. 效果评价

### 已改善

- 正确模板误报明显下降：首日累计基准规则不再仅凭“报告时间段首日”硬判，必须有表级首测证据或表内多数行支持。
- 深层/测斜统计摘要从硬错误改为复核提示，避免把正确模板里的统计口径差异判成 error。
- Paddle 异步提交有网络重试；重试前会重新定位文件指针，避免第二次上传空文件。
- Paddle 网络失败时不再重复尝试三套 profile，也不再转 legacy base64 接口。
- PDF 文本层增加 PyMuPDF 兜底入口，打包配置同时包含 `pymupdf` 和 `fitz`。
- Streamlit 已补齐 Excel 中间层下载，并通过浏览器交互验证真实下载。
- 桌面端会把本机旧设置里的 `PaddleOCR-VL` / `PaddleOCR-VL-1.5` 默认迁移显示为 `PaddleOCR-VL-1.6`，避免升级后继续展示旧默认模型；用户仍可在下拉框里手动选择其它模型。

### 仍需人工关注

- LLM 分块结构化是主要耗时来源；100 页级 PDF 仍可能需要 5 到 8 分钟。
- `deepseek-v4-flash` 的大表格结构化结果存在波动，同一模板不同运行会出现表数差异；Excel 中间层是当前最重要的复核入口。
- 错误版模板的召回率还不能只看 error 数，需要和逐字段 ground truth 对齐后才能给出准确检出率。
- PaddleOCR 外部服务当前连接失败，无法证明远程 OCR 全链路可用，只能证明失败处理路径不会闪退或退化为 MemoryError。
